# Track 3.19 NEXUS Import Adapter Report Contract

## Purpose

Track 3.19 defines a stable local report contract for the NEXUS import adapter.

The contract hardens successful single-run reports, regression-suite reports, fail-closed failure reports, and stop-condition reports. It does not create a public capability and does not change public site behavior.

## Relationship to Tracks 3.17 and 3.18

Track 3.17 created the first local-only adapter run against the pinned NEXUS source.

Track 3.18 created a local-only regression suite across all five current AETHERUS fixtures.

Track 3.19 standardizes the report and failure contract those local scripts must preserve before any future adapter expansion.

## Report Types

- Single-run success report: generated by `scripts/run-nexus-import-adapter-local.mjs`.
- Regression-suite report: generated by `scripts/run-nexus-import-adapter-regression-suite.mjs`.
- Fail-closed failure report: generated when the adapter cannot safely execute or normalize.
- Stop-condition report: generated when deterministic identity, pinned source, release eligibility, trace, or claim-boundary checks fail.

## Required Fields

Single-run reports must include metadata, NEXUS boundary, source, adapter input, NEXUS payload, raw result boundary, normalized interface result, release eligibility, trace boundary, claim boundary, deterministic identity, and stop conditions.

Regression reports must include metadata, NEXUS boundary, suite summary, fixture results, deterministic identity summary, release eligibility summary, trace boundary summary, claim boundary, and stop conditions.

Failure reports must include metadata, failure category, failure reason, source, attempted fixture id, NEXUS boundary, normalized verdict, release eligibility, trace boundary, claim boundary, stop conditions, and raw error boundary.

## Failure Categories

Allowed failure categories are:

- `nexus_path_missing`
- `nexus_commit_mismatch`
- `nexus_working_tree_dirty`
- `fixture_mapping_missing`
- `regulatory_context_missing`
- `manifest_mapping_missing`
- `nexus_execution_failure`
- `malformed_nexus_result`
- `unknown_nexus_verdict`
- `unknown_risk_level`
- `missing_omega_decision`
- `nondeterministic_output`
- `release_eligibility_incoherent`
- `trace_boundary_violation`
- `claim_boundary_violation`

## Release Eligibility Invariants

A `fail` or `escalate` verdict must never be release-eligible.

Unsupported mapping, local execution failure, and nondeterministic decision-relevant output must never be release-eligible.

JSONL audit-log output must not imply ledger validity, operational audit evidence, persistence, or release authority.

## Trace / Ledger Boundary

Local adapter trace-like events must use:

```text
local_adapter_run_not_persistent_not_ledger
```

NEXUS JSONL output is local audit-log output only. It is not a persistent ledger, immutable ledger, hash-chain ledger, database trace store, or production audit record.

## Claim Boundary

Allowed after Track 3.19:

- “local import-adapter report contract”
- “fail-closed adapter report schema”
- “local adapter regression reports validated”

Forbidden after Track 3.19:

- “AETHERUS_MONOLITH runs NEXUS publicly”
- “NEXUS powers the public interface”
- “NEXUS-integrated live system”
- “live governance kernel”
- “persistent NEXUS ledger”
- “production runtime”
- “enterprise deployment”

## Command

```bash
node scripts/validate-nexus-import-adapter-reports.mjs
```

## What This Does Not Validate

- No public integration.
- No live site behavior.
- No backend.
- No persistence.
- No ledger.
- No authentication.
- No model execution.
- No live orchestration.
- No enterprise deployment.
- No compliance certification.
