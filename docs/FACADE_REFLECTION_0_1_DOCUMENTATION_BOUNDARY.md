# Facade Reflection 0.1 Documentation Boundary

## Purpose

Facade Reflection 0.1 explains the current local Conduit evidence chain from the perspective of the public static Facade.

AETHERUS_MONOLITH remains a static, GitHub Pages deployed, browser-side, deterministic, scenario-driven, non-persistent, prototype-facing, claim-bounded public artifact. The public Facade communicates governance architecture, deterministic scenario behavior, evidence vocabulary, and claim boundaries. It does not expose the local Track 3 Conduit chain as public runtime capability.

Track 3 Conduit work is local-only evidence and integration-contract work. It defines contracts, fixtures, validators, stubs, report shapes, and ignored local report outputs that can support future review. It does not add public NEXUS execution, public UI wiring, backend routes, authentication, database behavior, persistence, ledger behavior, live orchestration, Palisade behavior, Weave behavior, alternate Vault execution, active Vault switching, or runtime multi-Vault support.

## Local vs Public Boundary

The public surface is the Facade: a static GitHub Pages artifact that runs deterministic browser-side scenario logic and keeps trace output in memory only.

The local surface is the Conduit evidence chain: repository contracts, fixture-backed validators, metadata-only stubs, and ignored `.track3-runs/` reports generated by local commands.

Local Conduit evidence may describe compatibility checks, failure handling, report validation, and pinned Vault metadata. It must not be presented as public runtime capability, production deployment, enterprise deployment, customer deployment, compliance certification, persistent audit infrastructure, or public NEXUS execution.

## Claim Language

### Currently claimable

- AETHERUS_MONOLITH is a static, GitHub Pages deployed, browser-side, deterministic, scenario-driven, non-persistent, prototype-facing, claim-bounded public artifact.
- The public Facade can present governance architecture, deterministic scenario traces, evidence vocabulary, and claim boundaries.
- Track 3 Conduit artifacts provide local-only contracts, fixtures, validators, metadata-only stubs, and ignored local reports for integration-contract evidence.
- The current local Vault evidence pin is the NEXUS MVP source at `ab95cbbd24df5817c4e363d24b3b199ac8af6c6f`.
- Palisade is not instantiated.
- Weave is specification only and not runnable.

### Not yet claimable

- Public NEXUS runtime execution.
- Public browser-side adapter calls.
- Backend routes, hosted services, authentication, authorization, billing, database behavior, storage, persistence, or telemetry.
- Persistent ledger behavior or database-backed trace storage.
- Production SaaS readiness, enterprise deployment, customer deployment, or operational decision-making.
- Palisade runtime behavior or enforcement.
- Weave runtime behavior or orchestration.
- Alternate Vault execution, active Vault switching, or runtime multi-Vault support.
- Compliance certification, certified control operation, or independent assurance.

### External event claims

External event claims require external evidence before they can be made. Certification claims require certification evidence. Assessment claims require assessment evidence. Deployment claims require deployment evidence. Customer claims require customer evidence. Audit claims require audit evidence. Operational claims require operational runtime evidence and review of the relevant security, reliability, governance, privacy, and legal boundaries.

Until that evidence exists and is accepted, the public language should remain bounded to the static Facade and local-only Conduit evidence chain described above.
